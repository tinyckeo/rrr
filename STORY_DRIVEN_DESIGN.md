# 故事驅動型設計架構

## 設計理念
通過真實家庭故事和寶寶成長日記，讓父母在故事中自然發現貝親產品的價值，建立情感連接而非硬性推銷。

## 核心故事線

### 1. 家庭故事（Family Stories）
展示不同家庭背景的真實經歷：
- **新手媽媽的第一次副食品冒險**（林媽媽，寶寶6個月）
- **職業媽媽的外出用餐解決方案**（王媽媽，寶寶10個月）
- **雙寶家庭的進食訓練日常**（陳媽媽，雙寶1-2歲）
- **爺爺奶奶的照顧貼心選擇**（李奶奶，孫子1.5歲）

### 2. 寶寶成長時間軸（Growth Timeline）
按月齡呈現關鍵時刻：
- 4-6個月：開始副食品
- 6-8個月：嘗試更多食物
- 8-10個月：學習自主進食
- 10-12個月：獨立進食訓練
- 12-18個月：多樣化飲食
- 18-24個月：完全自主進食

### 3. 故事元素
每個故事包含：
- **故事標題和簡介**
- **家庭背景**（媽媽名字、寶寶年齡、家庭情況）
- **故事敘述**（3-4段文字，描述真實場景和挑戰）
- **關鍵時刻圖片**（故事中的真實照片）
- **使用的產品**（自然融入故事中的產品）
- **媽媽心得**（使用產品的感受）

## 頁面結構

### 首頁
1. Hero Banner - 溫暖的家庭場景
2. 故事精選 - 3-4個特色故事卡片
3. 成長時間軸預覽 - 按月齡展示故事
4. 品牌故事 - 貝親的使命
5. 頁腳

### 故事詳情頁
1. 故事 Hero 圖片
2. 家庭介紹卡片
3. 完整故事敘述（分段落）
4. 故事中的關鍵時刻（圖片+文字）
5. 使用的產品推薦
6. 相關故事推薦

### 成長時間軸頁
1. 互動式時間軸（可點擊各月齡）
2. 該月齡的故事列表
3. 推薦產品
4. 育兒知識小貼士

## 設計特色

### 視覺風格
- 溫暖、親切、真實
- 使用真實家庭照片而非模特
- 柔和的色彩過渡
- 手寫風格的標題和標籤

### 互動體驗
- 故事卡片 Hover 效果
- 時間軸滾動動畫
- 平滑的頁面過渡
- 故事進度指示

### 內容策略
- 強調媽媽的真實聲音
- 展示產品在日常生活中的應用
- 建立社群感（多個家庭故事）
- 教育性內容（育兒知識）

## 故事數據結構

```typescript
interface Story {
  id: string;
  title: string;
  subtitle: string;
  familyName: string;
  momName: string;
  babyAge: string;
  babyName: string;
  familyBackground: string;
  heroImage: string;
  storyContent: string;
  keyMoments: Array<{
    image: string;
    caption: string;
  }>;
  products: Array<{
    id: string;
    name: string;
    role: string; // 在故事中的角色
  }>;
  momThoughts: string;
  relatedStories: string[];
  ageRange: string;
  category: 'first-time' | 'working-mom' | 'multiple-kids' | 'grandparents';
}
```

## 故事分類

| 分類 | 描述 | 目標受眾 |
|------|------|---------|
| 新手媽媽 | 第一次接觸副食品和餐具 | 0-1歲寶寶的新手父母 |
| 職業媽媽 | 外出、時間管理、便利性 | 需要便利解決方案的家庭 |
| 多寶家庭 | 同時照顧多個孩子 | 有多個孩子的家庭 |
| 祖父母照顧 | 代間照顧的溫暖故事 | 由祖父母照顧的孩子 |
